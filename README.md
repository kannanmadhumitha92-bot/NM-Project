# Cyber Security Project

## Project Title

Enforcing Least-Privilege Access Through Identity-Aware Application Policies Using Zscaler Private Access (ZPA)

## Project Overview

This project focuses on implementing Zero Trust Network Access (ZTNA) using Zscaler Private Access (ZPA) to enforce least-privilege access for users. In traditional networks, users are often given broad access once they connect to the network. However, this creates security risks.

Using identity-aware access policies, ZPA ensures that users can only access the specific applications they are authorized to use. Access is granted based on user identity, device posture, and security policies, reducing the risk of unauthorized access.

The project also explains how Identity Providers (IdPs) such as Azure Active Directory, Okta, and Active Directory integrate with ZPA to provide secure authentication and authorization.

## Project Stages

1. Understanding Zero Trust Security Model

Study the Zero Trust architecture, which follows the principle of “Never Trust, Always Verify.” Every user and device must be verified before accessing applications.

2. Identity Provider Integration

Integrate Identity Providers (IdPs) like Azure AD, Okta, or Active Directory with Zscaler Private Access to authenticate users securely.

3. Application Access Policy Configuration

Create identity-aware and application-specific access policies to ensure users can access only the applications they are permitted to use.

4. Secure User-to-Application Connectivity

Use ZPA connectors to securely connect users to private applications without exposing them to the internet or internal network.

## Tools Used

-Zscaler Private Access (ZPA) – For Zero Trust application access
-Azure Active Directory / Okta / Active Directory – Identity authentication
-Browser Developer Tools – For analyzing application access behavior
